% 1. INTRODUCTION
% -------------------------------------------------------------
\chapter{Introduction}

The Fleet Management System is a desktop-based Java application designed to help track, manage and rent different types of vehicles such as Cars, Bikes and Trucks. The system streamlines the process of adding vehicles, marking them as rented or returned and maintaining an organised local database using text file storage.

This project emphasises Object-Oriented Programming principles including inheritance, polymorphism, abstraction and the use of interfaces. A Swing-based Graphical User Interface (GUI) enables easy interaction with the system, and CSV-based import and export features support data portability. The project is suitable for small rental-based businesses and serves as an excellent academic demonstration of Java programming concepts.

% 3. PROPOSED SYSTEM
% -------------------------------------------------------------
\chapter{Proposed System}

The proposed system digitalises rental fleet operations using a structured software approach. It eliminates manual record-keeping and provides consistent handling of all vehicle types via a unified interface. 

Key functionalities:
\begin{itemize}
    \item Add new vehicles to the fleet
    \item Rent and return vehicles with real-time status updates
    \item Import and export fleet data through CSV
    \item GUI interface for simplified interaction
    \item Persistent file-based storage
\end{itemize}

The design offers reliability, clarity and extensibility for future upgrades.

% -------------------------------------------------------------
% 4. SYSTEM DESIGN
% -------------------------------------------------------------
\chapter{System Design}

\section*{Architecture}

The system follows a Model–View–Controller (MVC) structure:
\begin{itemize}
    \item \textbf{Model}: Vehicle, Car, Bike, Truck, Rentable, VehicleNotAvailableException
    \item \textbf{Service}: FleetService (business logic)
    \item \textbf{View}: FleetGUI (Java Swing based)
\end{itemize}

\section*{Class Description}
\textbf{Vehicle}\\
Base class with fields ID, Brand and rental status.

\textbf{Car, Bike, Truck}\\
Derived classes implementing the Rentable interface. Each overrides getTypeName() and getExtra().

\textbf{FleetService}\\
Handles add, rent, return, search, import, export and persistence.

\textbf{FleetGUI}\\
Provides the visual interface with table view, search bar and input form.
